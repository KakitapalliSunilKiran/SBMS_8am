What is ResponseEntity?

ResponseEntity is a Spring class that represents an HTTP response.
It allows you to control:

✔ Response body
✔ HTTP status code
✔ HTTP headers

So instead of just returning data, you return a complete HTTP response.

📌 Why do we use ResponseEntity?

Because a REST API should return proper HTTP responses like:

200 OK

201 CREATED

404 NOT FOUND

500 INTERNAL SERVER ERROR

Using ResponseEntity, you can specify these statuses and messages yourself.

🌟 Basic Syntax
ResponseEntity<T>


Where T is the type of the response body, such as:

ResponseEntity<Student>

ResponseEntity<String>

ResponseEntity<List<Student>>

ResponseEntity<Void>

🧠 Example 1: Returning data with status
return ResponseEntity.ok(student);


This returns:

Body: student

Status: 200 OK

🧠 Example 2: Returning a custom HTTP status
return new ResponseEntity<>(student, HttpStatus.CREATED);


This returns:

Body: student

Status: 201 CREATED

🧠 Example 3: Returning error message
return ResponseEntity
        .status(HttpStatus.NOT_FOUND)
        .body("Student not found");


This sends:

Body: "Student not found"

Status: 404 NOT FOUND

🧠 Example 4: Sending no body (empty response)
return ResponseEntity.noContent().build();


This returns:

204 NO CONTENT

=========================================================
You can integrate ResponseEntity with headers very easily in Spring.
Headers allow you to send additional information back to the client such as:

✔ Custom metadata
✔ Authorization info
✔ Pagination details
✔ Timestamps
✔ Content type hints

⭐ 1. Adding Headers with ResponseEntity
Example:
@GetMapping("/hello")
public ResponseEntity<String> sayHello() {

    HttpHeaders headers = new HttpHeaders();
    headers.add("App-Version", "1.0");
    headers.add("Author", "Sunil");

    return new ResponseEntity<>(
            "Hello World!",
            headers,
            HttpStatus.OK
    );
}

Response sent:

Body: "Hello World!"

Status: 200 OK

Headers:
